# KR-JP Match Database ERD

## Tables

### users
- Primary key: `id`
- Stores account identity, nationality, language, and verification state

### profiles
- Primary key / foreign key: `user_id -> users.id`
- One-to-one extension table for age, bio, and interests

### matches
- Primary key: `id`
- Foreign keys:
  - `user1_id -> users.id`
  - `user2_id -> users.id`
- Stores one unique user pair and their matching score

### messages
- Primary key: `id`
- Foreign keys:
  - `sender_id -> users.id`
  - `receiver_id -> users.id`
- Stores original and translated chat content

### profile_photos
- Primary key: `id`
- Foreign key:
  - `user_id -> users.id`
- Stores uploaded profile image metadata

### refresh_sessions
- Primary key: `id`
- Foreign key:
  - `user_id -> users.id`
- Stores refresh-token session rotation state

### reports
- Primary key: `id`
- Foreign keys:
  - `reporter_id -> users.id`
  - `reported_id -> users.id`
- Stores user safety reports

## Relationship Summary

- `users 1:1 profiles`
- `users 1:N messages` as sender
- `users 1:N messages` as receiver
- `users 1:N reports` as reporter
- `users 1:N reports` as reported
- `users 1:N profile_photos`
- `users 1:N refresh_sessions`
- `users N:N users` through `matches`

## Mermaid ERD

```mermaid
erDiagram
    USERS ||--|| PROFILES : has
    USERS ||--o{ MESSAGES : sends
    USERS ||--o{ MESSAGES : receives
    USERS ||--o{ REPORTS : files
    USERS ||--o{ REPORTS : is_reported_in
    USERS ||--o{ MATCHES : user1
    USERS ||--o{ MATCHES : user2
    USERS ||--o{ PROFILE_PHOTOS : uploads
    USERS ||--o{ REFRESH_SESSIONS : rotates

    USERS {
        uuid id PK
        text email
        text password_hash
        text gender
        text nationality
        text language
        boolean is_verified
        timestamptz created_at
    }

    PROFILES {
        uuid user_id PK, FK
        int age
        text age_group
        text bio
        text_array interests
    }

    MATCHES {
        uuid id PK
        uuid user1_id FK
        uuid user2_id FK
        numeric score
        timestamptz created_at
    }

    MESSAGES {
        uuid id PK
        uuid sender_id FK
        uuid receiver_id FK
        text original_text
        text translated_text
        text translation_status
        timestamptz translated_at
        timestamptz read_at
        text language_from
        text language_to
        timestamptz created_at
    }

    PROFILE_PHOTOS {
        uuid id PK
        uuid user_id FK
        text file_url
        int display_order
        timestamptz created_at
    }

    REFRESH_SESSIONS {
        uuid id PK
        uuid user_id FK
        text token_jti
        timestamptz issued_at
        timestamptz expires_at
        timestamptz revoked_at
    }

    REPORTS {
        uuid id PK
        uuid reporter_id FK
        uuid reported_id FK
        text reason
        timestamptz created_at
    }
```
